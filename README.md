## Development Workflow

This package uses the standard devtools workflow. After cloning:

```r
# Install development dependencies
install.packages("pak")
pak::pak(c("devtools", "roxygen2", "testthat", "lintr", "styler"))

# Then, for day-to-day development:
devtools::load_all()    # Load the package into your session
devtools::document()    # Regenerate NAMESPACE and man/ from roxygen comments
devtools::test()        # Run all tests
devtools::check()       # Full R CMD check — must pass before committing
```

## Adding a Function

1. Create `R/my_function.R` — write your function with a full roxygen2 block and `@export`
2. `devtools::document()` — regenerates `NAMESPACE` and `man/my_function.Rd`
3. Create `tests/testthat/test-my_function.R` — write tests covering happy path and error cases
4. `devtools::check()` — must return **0 errors | 0 warnings | 0 notes** before committing

See [R/hello.R](R/hello.R) for a complete example of the expected pattern.

## Dependencies

Declare all runtime dependencies in `DESCRIPTION` under `Imports`.
Use `pak::pak("package-name")` to install during development.
Never use `library()` inside package code — use namespace-qualified calls (`pkg::fn()`)
or declare imports via roxygen2 (`@importFrom pkg fn`).
